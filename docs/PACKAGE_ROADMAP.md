# Package Roadmap

## Current foundation packages

| Package | Purpose |
| --- | --- |
| `@ui-base/core` | Shared base class, common attributes, events, localization, validation, utilities. |
| `@ui-base/design-system` | Design principles, accessibility standard, tokens, component spec template, lifecycle docs. |
| `@ui-base/theme` | Default, dark, and sample-tour theme CSS variable overrides. |
| `@ui-base/icons` | Embedded SVG registry, URL icons, and `uib-icon`. |
| `@ui-base/ui` | Controls, label/help, navigation, layout, overlay stubs, and tour action components. |
| `@ui-base/forms` | Form container, form control stubs, validation model, form layout stubs. |
| `@ui-base/calendar` | Calendar views. |
| `@ui-base/hero` | Hero component and defaults. |

## Future packages

| Package | Purpose |
| --- | --- |
| `@ui-base/testing` | Accessibility checks, keyboard tests, mock data, component test helpers. |
| `@ui-base/cli` | Future `mt new`, `mt add`, `mt upgrade`, `mt doctor`, and component scaffolding commands. |
| `@ui-base/maps` | Map and geospatial components. |
| `@ui-base/charts` | Chart and dashboard components. |

## Stub policy

When a component is part of the roadmap but not ready for full production use, the workspace includes a lightweight experimental stub. This lets app teams see the API direction early without assuming the component is stable.
